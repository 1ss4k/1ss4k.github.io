1、349.两个数组的交集

题意：给定两个数组，编写一个函数来计算它们的交集

思路：每个数组把其中的数字维护一个表/检查两个表中相同的部分。

实现：

自己不会实现...直接参考别人的

参考答案

class Solution { //这个是题目固定的结构

public:
vector<int> intersection(vector<int>& nums1, vector<int>& nums2) {
    unordered_set<int> result_set; // 存放结果

    unordered_set<int> nums_set(nums1.begin(), nums1.end()); //相当于把num1存放在nums_set中去了。

    for (int num : nums2) {
        // 发现nums2的元素 在nums_set里又出现过
        if (nums_set.find(num) != nums_set.end()) {
            result_set.insert(num);
        }
    }
    return vector<int>(result_set.begin(), result_set.end());

} //end intersection;

}; // end Solution;

收获：

1、使用到的数据结构：unordered_set。set可以排除重复的元素。set和multiset的底层实现是红黑树，unordered_set的底层实现是哈希表。
/
vector是一个容器，vector& nums1, vector& nums2是把地址传递进去了，如果修改的话可以改变数组中他们的真实的值。

2、最核心的部分是中间那个for循环，就是遍历num2中的每个元素，如果能在num1中找到，就说明是共同出现的，就把结果放在result中即可。

2、242.有效的字母异位词

题意：给定两个字符串 s 和 t ，编写一个函数来判断 t 是否是 s 的字母异位词。

思路：：把s中的字符都记录下来，再把t中的字符都记录下来，判断两个是否相等。

实现：
还是没实现。

参考答案：

1、暴力实现
两层for循环分别检测，还要注意是否有重复，不去实现了。

2、哈希表来实现。

数组其实就是一个简单的哈希表。可以定义一个数组来记录单词中每个字母出现的次数。并且由于只是判断两个字符的，只要在一个数组上记录次数就行，一个加一个减，最后只要判断是不是数组的每个字符的次数都是0就好。

class Solution {
public:
bool isAnagram(string s, string t) { //主函数
    int record[26] = {0}; 
    for (int i = 0; i < s.size(); i++) {
        // 并不需要记住字符a的ASCII，只要求出一个相对数值就可以了
        record[s[i] - 'a']++; 
    }
    for (int i = 0; i < t.size(); i++) {
        record[t[i] - 'a']--;
    }
    for (int i = 0; i < 26; i++) {
        if (record[i] != 0) {
            // record数组如果有的元素不为零0，说明字符串s和t 一定是谁多了字符或者谁少了字符。
            return false;
        }
    }
    // record数组所有元素都为零0，说明字符串s和t是字母异位词
    return true;
} //end isAnagram
}; //end Solution;

收获：

1、string字符串，可以使用str.size()函数来求得字符串的长度。

2、由于不需要求得确定的ascii，只需要一个相对的位置即可，可以用s[i]-‘a’来求得对于a的偏移。

3、这种方法可以解决特殊的ascii码的字符的情况。

3、383. 赎金信

1、题意：
给定一个赎金信 (ransom) 字符串和一个杂志(magazine)字符串，判断第一个字符串 ransom 能不能由第二个字符串 magazines 里面的字符构成。如果可以构成，返回 true ；否则返回 false。

2、思路：

最开始写出的代码如下 

class Solution {
public:
bool canConstruct(string ransomNote, string magazine) {
    int record[26]={0}; //最开始这个忘记了置0
    for(int i = 0; i < magazine.size(); i++){
        record[magazine[i]-'a']++;
    }
    for(int i = 0; i < ransomNote.size(); i++){
        record[ransomNote[i] - 'a']--;
    }
    for(int i = 0; i <= 25; i++){
        if(record[i] < 0) return false;
    }
    return true;
}
};

本开始报错，说是结果错了。因为最后i循环的时候没有包括进去25，所以有z的测试用例就会出错。

3、参考答案

这种方法是很好的，复杂度也比较低。另外还有暴力破解的方法，效果不是很好，但是这边也来看一下。

class Solution {
public:
bool canConstruct(string ransomNote, string magazine) {
    for (int i = 0; i < magazine.length(); i++) {
        for (int j = 0; j < ransomNote.length(); j++) {
            // 在ransomNote中找到和magazine相同的字符
            if (magazine[i] == ransomNote[j]) { 
                ransomNote.erase(ransomNote.begin() + j); // ransomNote删除这个字符
                break;
            }
        }
    }
    // 如果ransomNote为空，则说明magazine的字符可以组成ransomNote
    if (ransomNote.length() == 0) { 
        return true;
    }
    return false;
}
};

思想就是：如果在magazine中找到了某一个字符是存在于ransomNote中的，就把ransomeNote中的这个给字符删除掉。最后看一下ransomeNote中的字符是不是全部都被删除掉了。
/但是它的时间复杂度是比较高的，而且erase的复杂度也比较高。
